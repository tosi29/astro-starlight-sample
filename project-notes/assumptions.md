# 不明点・仮定・設計判断

## 仮定

- GitHub Pagesの公開URLは `https://tosi29.github.io/astro-starlight-sample/` とした。
- `docs/` は生成物として扱い、ソースは `src/`、設計メモは `project-notes/` に置く。
- このPRではレビュー差分を小さくするため、`docs/` はコミットせず `.gitignore` に含める。
- 評価対象は詳細レポートではなく、人間が触って確認できるサンプルサイトとした。
- 30ページ程度の要件は、トップ・Feature Tour・カテゴリ別ページを含む31ページで満たす。

## 設計判断

- Starlightの標準機能を優先し、過度なカスタム実装は避けた。
- AIログ、技術メモ、調査メモ、ADR、用語集を別章に分け、サイドバーが破綻しないか確認しやすくした。
- MDXページを含め、Starlightコンポーネントと独自Astroコンポーネントの利用感を確認できるようにした。
- `docs/` はビルド成果物コミットとして分ける前提にした。
