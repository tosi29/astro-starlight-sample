<Aside type="tip" title="使い方">
  まずこのページを開き、気になる観点から対応ページへ移動してください。詳細な評価レポートではなく、人間が触って判断するための案内です。
</Aside>

## 確認観点マップ

| 観点 | 確認ページ | 見るポイント |
| :--- | :--- | :--- |
| サイドバー | サイト構造、各カテゴリ | 30ページ規模で迷わないか |
| 検索 | 用語集、FAQ、調査メモ | 日本語語句で探しやすいか |
| 目次 | 長めの技術メモ、ADR | h2/h3の見通しが良いか |
| Markdown | 比較表、コード、引用 | 技術メモとして十分か |
| MDX | コンポーネント実験 | カード、タブ、独自部品が使えるか |
| AIログ再編集 | before/afterページ | 会話ログを記事化しやすいか |
| GitHub Pages | README、公開手順 | `docs/`公開が現実的か |

## 触り方

<Tabs>
  <TabItem label="初見読者">
    トップ、サイト構造、AIログ再編集の順に読み、読者導線が作れるか確認します。
  </TabItem>
  <TabItem label="運用者">
    README、公開手順、編集フロー、ADRを見て、更新作業が重すぎないか確認します。
  </TabItem>
  <TabItem label="技術評価">
    技術メモ、MDX、コード表示、表、画像のページを重点的に確認します。
  </TabItem>
</Tabs>

## 代表的な確認ルート

1. トップページで目的を確認する
2. サイドバーから各章の深さを見る
3. 検索で「ADR」「Quartz」「RAG」「用語集」を探す
4. 長いページの右側目次が役立つか確認する
5. GitHub Pages公開手順をREADMEと照合する
